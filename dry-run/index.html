1. What is a dry run?

A dry run is the process of executing code on paper, in your mind, or in a table without actually running it on a computer. You read each line, update the values of variables, check conditions, and record output.

Think of it like rehearsing a play before the real performance. The code has not been executed by the machine, but you simulate it carefully.

2. Why is it important?

  • Builds logic: you understand what each line does.
  • Finds bugs early: wrong conditions and wrong updates become visible.
  • Improves confidence: you stop guessing and start reasoning.
  • Helps in interviews: many coding questions require careful tracing.
  • Makes debugging easier: you can locate the exact wrong step.

3. Dry run methodology

1

Read the full code once

Understand the overall purpose before tracing.

2

List the variables

Write down names like i, sum, count, a, and result.

3

Follow line by line

Execution order matters. Do not skip.

4

Update values immediately

Whenever a variable changes, record it at once.

5

Check conditions carefully

Every if and loop condition must be judged at that exact moment.

6

Track output separately

Keep printed values separate from variable values.

5. Golden rules

  • Never skip an iteration.
  • Never mix old and new values.
  • Update after each assignment.
  • Track output separately.
  • For loops, note the loop variable each time.
  • For arrays and lists, record changes after every append or update.

7. Example: collections

a = [] a.append(10) a.append(20) print(a)

Dry run:

  • Start with a = []
  • After a.append(10), a = [10]
  • After a.append(20), a = [10, 20]
  • print(a) outputs [10,20] in this visualizer format
