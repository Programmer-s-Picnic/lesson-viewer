1. What is JSON?

JSON means JavaScript Object Notation. It is a simple text format used to store and transfer data.

When an Angular application talks to a server, the server usually sends JSON. Angular receives that JSON, converts it into JavaScript objects, and displays it in the HTML template.

JSON is data

It contains names, numbers, lists, booleans, and nested objects.

Angular shows data

Angular receives JSON and uses data binding to show it on the page.

JSON Concept Example Angular Usage
Object { "name": "Angular" } Single course, user, product, post
Array [{...}, {...}] List of courses, users, products
String "Champak" Names, titles, descriptions
Number 999 Price, marks, id, count
Boolean true Published, available, active
Nested object { "author": { "name": "Roy" } } Address, profile, author, metadata
This lesson uses modern standalone Angular patterns. For HTTP requests, Angular’s official documentation now shows provideHttpClient() in application providers, and the HTTP API is provided by HttpClient from @angular/common/http.

3. JSON Basics

JSON looks like JavaScript object syntax, but it is stricter.

Valid JSON
{
  "id": 1,
  "title": "Angular JSON Processing",
  "price": 499,
  "published": true,
  "tags": ["Angular", "JSON", "Frontend"],
  "author": {
    "name": "Champak Roy",
    "brand": "Programmer's Picnic"
  }
}

Important JSON Rules

  • Keys must be inside double quotes.
  • String values must be inside double quotes.
  • No comments are allowed inside pure JSON.
  • No trailing comma is allowed after the last item.
  • JSON can contain object, array, string, number, boolean, or null.
Common mistake: This is invalid JSON because of the trailing comma: { "name": "Angular", }

8. Enable HttpClient

Angular applications use HttpClient to fetch JSON over HTTP. In a standalone Angular app, provide it once in app.config.ts.

src/app/app.config.ts
import { ApplicationConfig } from '@angular/core';
import { provideRouter } from '@angular/router';
import { provideHttpClient } from '@angular/common/http';

import { routes } from './app.routes';

export const appConfig: ApplicationConfig = {
  providers: [
    provideRouter(routes),
    provideHttpClient()
  ]
};
Official Angular documentation shows provideHttpClient() as the modern way to provide HttpClient in application providers.

Load local JSON using HttpClient

app.component.ts
import { Component, OnInit } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Course } from './models/course';

@Component({
  selector: 'app-root',
  imports: [],
  templateUrl: './app.component.html',
  styleUrl: './app.component.css'
})
export class AppComponent implements OnInit {
  courses: Course[] = [];

  constructor(private http: HttpClient) {}

  ngOnInit() {
    this.http.get<Course[]>('/data/courses.json')
      .subscribe(data => {
        this.courses = data;
      });
  }
}

9. Read JSON from an API

An API is a server endpoint that sends data. Angular can read API data in the same way it reads a local JSON file.

Example API call
this.http.get<Course[]>('https://example.com/api/courses')
  .subscribe(data => {
    this.courses = data;
  });

Using a GitHub Pages JSON URL

For your own static JSON files, you can keep JSON on GitHub Pages and read it in Angular.

Example
jsonUrl = 'https://programmer-s-picnic.github.io/json-images/links/users.json';

ngOnInit() {
  this.http.get<any[]>(this.jsonUrl)
    .subscribe(data => {
      console.log(data);
    });
}
CORS warning: If an API blocks browser requests, Angular cannot read it directly. The server must allow cross-origin requests.

10. Loading State and Error Handling

A professional app must show loading messages and errors.

app.component.ts
import { Component, OnInit } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Course } from './models/course';

@Component({
  selector: 'app-root',
  imports: [],
  templateUrl: './app.component.html',
  styleUrl: './app.component.css'
})
export class AppComponent implements OnInit {
  courses: Course[] = [];
  loading = true;
  errorMessage = '';

  constructor(private http: HttpClient) {}

  ngOnInit() {
    this.http.get<Course[]>('/data/courses.json')
      .subscribe({
        next: data => {
          this.courses = data;
          this.loading = false;
        },
        error: error => {
          console.error(error);
          this.errorMessage = 'Could not load courses. Please try again.';
          this.loading = false;
        }
      });
  }
}
app.component.html
@if (loading) {
  <p>Loading courses...</p>
}

@if (errorMessage) {
  <p class="error">{{ errorMessage }}</p>
}

@if (!loading && !errorMessage) {
  @for (course of courses; track course.id) {
    <div class="course-card">
      <h2>{{ course.title }}</h2>
      <p>₹{{ course.price }}</p>
    </div>
  }
}

11. Search and Filter JSON Data

After loading JSON, users often want to search or filter it.

app.component.ts
import { Component, OnInit } from '@angular/core';
import { FormsModule } from '@angular/forms';
import { HttpClient } from '@angular/common/http';
import { Course } from './models/course';

@Component({
  selector: 'app-root',
  imports: [FormsModule],
  templateUrl: './app.component.html',
  styleUrl: './app.component.css'
})
export class AppComponent implements OnInit {
  courses: Course[] = [];
  searchText = '';
  selectedCategory = 'All';

  constructor(private http: HttpClient) {}

  ngOnInit() {
    this.http.get<Course[]>('/data/courses.json')
      .subscribe(data => {
        this.courses = data;
      });
  }

  get filteredCourses() {
    const text = this.searchText.toLowerCase();

    return this.courses.filter(course => {
      const matchesText =
        course.title.toLowerCase().includes(text) ||
        course.category.toLowerCase().includes(text);

      const matchesCategory =
        this.selectedCategory === 'All' ||
        course.category === this.selectedCategory;

      return matchesText && matchesCategory;
    });
  }
}

12. Sort JSON Data

Sorting means changing the order of displayed JSON items.

app.component.ts
sortByPriceLowToHigh() {
  this.courses = [...this.courses].sort((a, b) => a.price - b.price);
}

sortByPriceHighToLow() {
  this.courses = [...this.courses].sort((a, b) => b.price - a.price);
}

sortByTitle() {
  this.courses = [...this.courses].sort((a, b) =>
    a.title.localeCompare(b.title)
  );
}
app.component.html
<button (click)="sortByPriceLowToHigh()">
  Price: Low to High
</button>

<button (click)="sortByPriceHighToLow()">
  Price: High to Low
</button>

<button (click)="sortByTitle()">
  Sort by Title
</button>
Why use [...this.courses]? It creates a new array before sorting. This is cleaner and safer than mutating the original array directly.

14. Add JSON Data from a Form

In the browser, you can add a new object to an existing JSON array. This does not automatically save to the original JSON file, but it updates the current page state.

app.component.ts
newCourse: Course = {
  id: 0,
  title: '',
  category: '',
  price: 0,
  published: true

addCourse() {
  const courseToAdd: Course = {
    ...this.newCourse,

  this.courses = [courseToAdd, ...this.courses];

  this.newCourse = {
    id: 0,
    title: '',
    category: '',
    price: 0,
    published: true

15. Move JSON Logic to a Service

A service keeps your component clean. The component should focus on UI. The service should handle data loading.

Terminal

import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Course } from '../models/course';

@Injectable({
  providedIn: 'root'
})
export class CourseService {
  private jsonUrl = '/data/courses.json';

  constructor(private http: HttpClient) {}

  getCourses() {
    return this.http.get<Course[]>(this.jsonUrl);
  }
}
app.component.ts
import { Component, OnInit } from '@angular/core';
import { Course } from './models/course';
import { CourseService } from './services/course.service';

@Component({
  selector: 'app-root',
  imports: [],
  templateUrl: './app.component.html',
  styleUrl: './app.component.css'
})
export class AppComponent implements OnInit {
  courses: Course[] = [];
  loading = true;
  errorMessage = '';

  constructor(private courseService: CourseService) {}

  ngOnInit() {
    this.courseService.getCourses()
      .subscribe({
        next: data => {
          this.courses = data;
          this.loading = false;
        },
        error: error => {
          console.error(error);
          this.errorMessage = 'Could not load courses.';
          this.loading = false;
        }
      });
  }
}
Professional habit: Keep HTTP logic in services. Keep display logic in components.

Step 4: Component Logic

src/app/app.component.ts
import { Component, OnInit } from '@angular/core';
import { FormsModule } from '@angular/forms';
import { HttpClient } from '@angular/common/http';
import { Course } from './models/course';

@Component({
  selector: 'app-root',
  imports: [FormsModule],
  templateUrl: './app.component.html',
  styleUrl: './app.component.css'
})
export class AppComponent implements OnInit {
  courses: Course[] = [];
  loading = true;
  errorMessage = '';

  searchText = '';
  selectedCategory = 'All';
  sortMode = 'title';

  constructor(private http: HttpClient) {}

  ngOnInit() {
    this.loadCourses();
  }

  loadCourses() {
    this.loading = true;
    this.errorMessage = '';

    this.http.get<Course[]>('/data/courses.json')
      .subscribe({
        next: data => {
          this.courses = data;
          this.loading = false;
        },
        error: error => {
          console.error(error);
          this.errorMessage = 'Unable to load course JSON.';
          this.loading = false;
        }
      });
  }

  get categories() {
    const categorySet = new Set(this.courses.map(course => course.category));
    return ['All', ...Array.from(categorySet)];
  }

  get filteredCourses() {
    const text = this.searchText.toLowerCase().trim();

    let result = this.courses.filter(course => {
      const matchesSearch =
        course.title.toLowerCase().includes(text) ||
        course.category.toLowerCase().includes(text) ||
        course.level.toLowerCase().includes(text);

      const matchesCategory =
        this.selectedCategory === 'All' ||
        course.category === this.selectedCategory;

      return matchesSearch && matchesCategory;
    });

    if (this.sortMode === 'title') {
      result = result.sort((a, b) => a.title.localeCompare(b.title));
    }

    if (this.sortMode === 'low') {
      result = result.sort((a, b) => a.price - b.price);
    }

    if (this.sortMode === 'high') {
      result = result.sort((a, b) => b.price - a.price);
    }

    return result;
  }
}

17. Common Mistakes and Fixes

Mistake Reason Fix
JSON file shows 404 Wrong folder or URL Put file in public/data/file.json and read /data/file.json
No provider for HttpClient provideHttpClient() missing Add it in app.config.ts
Can't bind to ngModel FormsModule missing Add imports: [FormsModule]
Blank page after JSON load JSON shape does not match interface/template Check property names like title, price, category
API blocked CORS issue Fix server CORS or use a backend proxy
Cannot save to JSON file Static files are read-only in browser Use backend, database, or serverless function

19. Quick Quiz

1. What does JSON stand for?
JavaScript Object Notation.
2. Which Angular service reads JSON over HTTP?
HttpClient.
3. Where can we put local static JSON in modern Angular?
Inside the public folder, for example public/data/courses.json.
4. What is the use of an interface?
It defines the expected shape of JSON data and helps TypeScript catch mistakes.
5. Can Angular permanently save data into a static JSON file from the browser?
No. For permanent saving, use a backend, database, serverless function, or API.
