1. Introduction to Angular Components

A component is a reusable piece of user interface. In Angular, a component usually has TypeScript logic, HTML template, CSS style, and optional test code.

Simple meaning: A component is a small visible part of a page, like a header, card, menu, form, footer, or dashboard panel. 
Angular App
├── Header Component
├── Course Card Component
├── Student Form Component
└── Footer Component

2. Level 0 Meaning

Imagine a website as a house. A component is like a door, window, room, table, or switch. Instead of building the entire website as one huge file, Angular lets us build it from small parts.

Component Logic TypeScript file with data and functions.
Component View HTML file that shows the component.
Component Style CSS file that designs the component. 
Component = TypeScript + HTML + CSS

3. Why Components Matter

Components make Angular apps easier to build, reuse, test, and maintain.

Without Components With Components
One huge HTML file Small reusable UI parts
Repeated card code One card component used many times
Hard to debug Each component can be checked separately
Hard to scale Project can grow feature by feature

9. Component Logic

The TypeScript class stores data and methods used by the template.

course-card.ts

import { Component } from '@angular/core';

@Component({
  selector: 'app-course-card',
  templateUrl: './course-card.html',
  styleUrl: './course-card.css'
})
export class CourseCard {
  title = 'Angular Components';
  description = 'Learn how to build reusable UI pieces.';
  started = false;

  startCourse(): void {
    this.started = true;
  }
}

course-card.html

<article class="course-card">
  <h3>{{ title }}</h3>
  <p>{{ description }}</p>

  <button (click)="startCourse()">
    Start Course
  </button>

  @if (started) {
    <p>Course started. Good luck.</p>
  }
</article>

12. Send Events to Parent with output()

Use output when a child component should notify the parent about something.

course-card.ts

import { Component, input, output } from '@angular/core';

@Component({
  selector: 'app-course-card',
  templateUrl: './course-card.html',
  styleUrl: './course-card.css'
})
export class CourseCard {
  title = input('Untitled Course');
  selected = output<string>();

  selectCourse(): void {
    this.selected.emit(this.title());
  }
}

course-card.html

<article class="course-card">
  <h3>{{ title() }}</h3>

  <button (click)="selectCourse()">
    Select
  </button>
</article>

Parent Template

<app-course-card
  title="Angular Components"
  (selected)="onCourseSelected($event)">
</app-course-card>

<p>Selected: {{ selectedCourse }}</p>

Parent TypeScript

selectedCourse = '';

onCourseSelected(courseTitle: string): void {
  this.selectedCourse = courseTitle;
}

13. Two-way Binding with model()

Use model() when a child component should receive a value and also update it.

rating-box.ts

import { Component, model } from '@angular/core';

@Component({
  selector: 'app-rating-box',
  templateUrl: './rating-box.html',
  styleUrl: './rating-box.css'
})
export class RatingBox {
  rating = model(0);

  increase(): void {
    this.rating.update(value => value + 1);
  }

  decrease(): void {
    this.rating.update(value => Math.max(0, value - 1));
  }
}

rating-box.html

<div class="rating-box">
  <button (click)="decrease()">-</button>
  <strong>{{ rating() }}</strong>
  <button (click)="increase()">+</button>
</div>

Parent Template

<app-rating-box [(rating)]="courseRating"></app-rating-box>

<p>Parent rating: {{ courseRating }}</p>

Parent TypeScript

courseRating = 3;

15. Component Lifecycle

A component is created, displayed, checked, and destroyed. Lifecycle hooks let us run code at these moments.

Hook When It Runs Common Use
ngOnInit After component is initialized Load initial data
ngOnDestroy Before component is removed Cleanup timers/subscriptions

Example

import { Component, OnInit, OnDestroy } from '@angular/core';

@Component({
  selector: 'app-course-card',
  templateUrl: './course-card.html',
  styleUrl: './course-card.css'
})
export class CourseCard implements OnInit, OnDestroy {
  ngOnInit(): void {
    console.log('CourseCard initialized');
  }

  ngOnDestroy(): void {
    console.log('CourseCard destroyed');
  }
}

19. Common Component Mistakes

Mistake Problem Fix
Forgetting to import child component Selector is not recognized. Add child component to parent imports.
Using wrong selector Component does not appear. Copy selector from generated .ts file.
Wrong exported class name Import error. Use exact class name from generated file.
Too much logic in one component Hard to maintain. Split into smaller components and services.
Editing generated test imports blindly Tests fail. Make tests match standalone component imports.
Putting API logic in every component Repeated code. Move API calls to services.

Final Summary

Question Answer
What is an Angular component? A reusable UI building block with TypeScript, HTML, and CSS.
Which command creates a component? ng generate component component-name
How do we show a child component? Use its selector in the parent template and import it in the parent component.
How does parent send data to child? Use input() or @Input().
How does child send events to parent? Use output() or @Output().
How can a custom child support two-way binding? Use model().
What supports projected content? <ng-content>
What is a good structure? Put pages in pages and reusable UI in components or shared/components.
