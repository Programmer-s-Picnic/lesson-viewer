Quick start (the 3 commands you‚Äôll use most)

For a clean project setup, do this inside your project folder:

# 1) Create a virtual environment
python -m venv .venv

# 2) Activate it (see venv section for Windows/macOS/Linux)
# 3) Install packages
python -m pip install -U pip
python -m pip install requests

# Run your script
python app.py
Best practice: Prefer python -m pip instead of pip. It guarantees you‚Äôre using pip for the exact Python you intended.

The python command

python runs scripts, starts an interactive REPL, or runs modules with -m.

Check your Python

python --version
python -V

# Where is this python coming from?
python -c "import sys; print(sys.executable)"
python -c "import sys; print(sys.path)"

Run a script / run inline code

python app.py
python path/to/script.py

python -c "print('hello')"
python -c "import platform; print(platform.platform())"

Run a module with -m (very important)

# Run pip through the same python
python -m pip --version

# Start a simple web server (great for testing)
python -m http.server 8000

# Run unittest discovery
python -m unittest

# Run a package/module as a script
python -m yourpackage

Useful flags

Quality-of-life
  • -i : drop into interactive mode after running a script
  • -m : run a module
  • -c : run a one-liner
  • -q : quieter REPL startup (varies by version)
Debug/diagnostics
  • -X dev : developer mode (extra warnings)
  • -W default : warning control
  • -OO : strip docstrings + optimize (rarely needed)
Tip: If python doesn‚Äôt work on Windows, try py (see ‚ÄúWindows: py launcher‚Äù).

The pip command

pip installs and manages Python packages from package indexes (like PyPI) into your current environment.

Install, upgrade, uninstall

# Install packages
python -m pip install requests

# Install a specific version
python -m pip install "requests==2.32.3"

# Upgrade a package
python -m pip install -U requests

# Uninstall
python -m pip uninstall requests

Inspect what‚Äôs installed

python -m pip list
python -m pip show requests

# See dependency tree (needs install)
python -m pip install pipdeptree
pipdeptree

Freeze / export installed packages

# Outputs exact installed versions
python -m pip freeze

Install from files / URLs

# From a requirements file
python -m pip install -r requirements.txt

# Editable install (for local development)
python -m pip install -e .

# From a git repo (example)
python -m pip install "git+https://github.com/psf/requests.git"

Useful pip options

Common
  • -U / --upgrade
  • -r install from requirements file
  • --no-cache-dir when cache causes trouble
  • --upgrade-strategy eager (advanced)
Diagnostics
  • -v verbose mode
  • --version show pip version
  • debug for environment details
# Example diagnostics
python -m pip --version
python -m pip debug -v
Avoid: Running pip with system Python on Linux in a way that breaks OS packages. Use a virtual environment for projects.

Virtual environments (venv)

Virtual environments isolate dependencies per project so packages don‚Äôt conflict. The most common built-in tool is venv.

Create a venv

python -m venv .venv

Activate

Windows (PowerShell) 
.\.venv\Scripts\Activate.ps1

If scripts are blocked: run PowerShell as user and set policy:
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

macOS / Linux (bash/zsh) 
source .venv/bin/activate

Deactivate

deactivate

Confirm you‚Äôre using the right python/pip

python -c "import sys; print(sys.executable)"
python -m pip --version
Tip: Name your venv .venv and add it to .gitignore.

Requirements & reproducible installs

There are two common ways to track dependencies: simple requirements.txt and lock files (Poetry/uv/conda).

Simple approach (small projects)

# Save exact versions currently installed
python -m pip freeze > requirements.txt

# Install later
python -m pip install -r requirements.txt
Note: pip freeze captures everything installed (including transitive deps). For teams, consider a tool that separates "direct" vs "locked" dependencies.

Better approach with pip-tools (pin + lock)

# 1) Create requirements.in with your top-level deps:
# requests
# rich

python -m pip install pip-tools

# 2) Compile to a locked requirements.txt (pins versions)
python -m piptools compile requirements.in

# 3) Sync your env to exactly those versions
python -m piptools sync requirements.txt

More tools you‚Äôll actually use

pipx (install CLI apps safely)

Use pipx for Python-based command-line tools (so they don‚Äôt pollute project venvs). 

# Install pipx (once)
python -m pip install --user pipx
python -m pipx ensurepath

# Install a CLI tool globally-isolated
pipx install black
pipx install ruff
pipx install poetry

# Run without installing (handy)
pipx run httpie --help

pre-commit (auto-format and lint on commit)

python -m pip install pre-commit
pre-commit install

tox / nox (test automation across versions)

tox (classic) 
python -m pip install tox
tox
nox (Pythonic) 
python -m pip install nox
nox

Poetry (modern dependency management)

Poetry manages dependencies with a pyproject.toml and lock file. It‚Äôs popular for apps/libraries because installs are reproducible. 

# Install Poetry (recommended via pipx)
pipx install poetry

# Start a project
poetry new myproj
cd myproj

# Add dependencies
poetry add requests
poetry add --group dev pytest ruff

# Install (creates venv + installs)
poetry install

# Run commands inside the env
poetry run python -V
poetry run pytest
Tip: For teams, committing poetry.lock helps keep everyone‚Äôs environment consistent.

uv (fast Python package + venv tooling)

uv is a newer tool that focuses on speed for installs and environment management. Many people use it as a faster replacement for parts of pip/venv workflows.

Note: Commands can vary by uv version and workflow. If you use uv in a team, standardize on one approach and document it. 
# Typical patterns (examples; exact commands may differ by version)
# Create and use a venv, then install
uv venv
uv pip install -U pip
uv pip install requests

# Or sync dependencies from a lock/config (workflow-dependent)
# uv sync

Windows: the py launcher

On Windows, you may have multiple Python versions. The py launcher helps you pick one reliably. 

# List installed Pythons (often works)
py -0p

# Run a specific major/minor
py -3.12 --version
py -3.11 -m pip --version

# Create a venv with a specific python
py -3.12 -m venv .venv
py -3.12 -m pip install -U pip
Tip: If python is pointing to the wrong version, use py -3.12 explicitly.

Conda basics (when to use it)

Conda is popular in data science because it can manage non-Python dependencies (like native libraries). Use it if your stack needs complex compiled packages (NumPy/SciPy stacks, CUDA, etc.). 

# Create an environment
conda create -n myenv python=3.12

# Activate / deactivate
conda activate myenv
conda deactivate

# Install packages
conda install numpy pandas
# Or use pip inside conda env for pure-Python packages:
python -m pip install requests
Mixing conda and pip: Prefer conda packages first, then pip for the rest. Avoid random mixing if you want stability.

Debugging & diagnostics from the CLI

Tracebacks + more info

# Print full tracebacks for crashes (useful in some cases)
python -X faulthandler app.py

pdb (built-in debugger)

# Drop into debugger on crash
python -m pdb app.py

Time quick snippets

python -m timeit "sum(range(1000000))"

Profile performance

python -m cProfile -s cumulative app.py

Packaging basics (build a wheel)

If you‚Äôre building a library or distributing code, these commands matter.

# Install build tool
python -m pip install build

# Build sdist and wheel (outputs to dist/)
python -m build
Tip: Most modern Python packaging uses pyproject.toml as the central config file.

Troubleshooting (common errors)

1) ‚Äúpython/pip is not recognized‚Äù (Windows)

  • Try py instead of python.
  • Ensure Python is added to PATH during install.
  • Restart terminal after installing Python.

2) Installing goes to the wrong Python

# Always bind pip to the python you want:
python -m pip install PACKAGE

# Confirm:
python -m pip --version
python -c "import sys; print(sys.executable)"

3) Permission denied / cannot write

  • Use a virtual environment.
  • Or install to user site: python -m pip install --user PACKAGE

4) SSL / certificate errors

  • Often caused by corporate proxies or outdated certificate stores.
  • Try upgrading pip: python -m pip install -U pip
  • Check your network/SSL inspection setup if on a managed network.

5) ‚ÄúExternally managed environment‚Äù (some Linux distros)

Meaning: The OS protects the system Python. Use a venv for projects, or use pipx for CLI tools.
