Recommended workflow

Follow this order to reduce setup mistakes.

4 steps
1

Open capture.html

Enter a person name, start the camera, and save several clear face crops for that person.

2

Export training data

Use the export option to create faces.json. Download the saved training images as well.

3

Place files correctly

Put the images in images/faces/ and the exported JSON in data/faces.json.

4

Open recognize.html

Reload training if needed, start the camera, and verify that labels appear instead of only Unknown.

Help

What each page is for.

capture.html is the data collection tool. Use it before recognition.
recognize.html is the live recognition tool. Use it after training files are ready.
index.html is now the hub and should remain the main entry page.

Common checks

Things to verify when labels do not appear.

Make sure camera access is allowed and the page is served over a secure context.
Check that data/faces.json exists and its image paths match the actual files.
Use 3 to 5 clear face images per person with decent lighting and visible facial features.

Folder expectations

The recognition page expects these paths.

index.html → this hub page
capture.html → training image capture page
recognize.html → live recognition page
data/faces.json → grouped training metadata
images/faces/... → saved face crops referenced by the JSON file
