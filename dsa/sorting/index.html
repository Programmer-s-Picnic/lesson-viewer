Algorithms

Bubble Sort

O(n) / O(n²) / O(n²)

Repeatedly compares adjacent boxes and swaps them when they are out of order. Large values bubble toward the end.
Stable: YesIn-place: YesType: Core / Counting / Heap
Selection Sort

O(n²) / O(n²) / O(n²)

Selects the minimum from the unsorted portion and places it into the next sorted position.
Stable: NoIn-place: YesType: Core / Counting / Heap
Insertion Sort

O(n) / O(n²) / O(n²)

Builds the sorted part from left to right by inserting each new item into its correct position.
Stable: YesIn-place: YesType: Core / Counting / Heap
Merge Sort

O(n log n) / O(n log n) / O(n log n)

Splits the list into smaller parts, sorts both halves, then merges them back in order.
Stable: YesIn-place: NoType: Divide & Conquer
Quick Sort

O(n log n) / O(n log n) / O(n²)

Chooses a pivot, partitions smaller and larger values around it, then sorts the two parts recursively.
Stable: NoIn-place: UsuallyType: Divide & Conquer
Heap Sort

O(n log n) / O(n log n) / O(n log n)

Builds a max-heap, repeatedly removes the largest element, and places it at the end.
Stable: NoIn-place: YesType: Core / Counting / Heap
Counting Sort

O(n + k) / O(n + k) / O(n + k)

Counts how many times each value occurs, then rebuilds the sorted list from those counts.
Stable: YesIn-place: NoType: Core / Counting / Heap
Radix Sort

O(nk) / O(nk) / O(nk)

Sorts numbers digit by digit, usually from the least significant digit to the most significant digit.
Stable: YesIn-place: NoType: Core / Counting / Heap
Bucket Sort

O(n + k) / O(n + k) / O(n²)

Distributes values into buckets, sorts each bucket, and then joins the buckets together.
Stable: DependsIn-place: NoType: Core / Counting / Heap
TimSort

O(n) / O(n log n) / O(n log n)

Real-world hybrid that finds natural runs, uses insertion sort on small runs, and merges them efficiently.
Stable: YesIn-place: NoType: Divide & Conquer
