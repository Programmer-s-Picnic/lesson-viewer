Lesson 1: Maven, Gradle, IDE choice, and installation

Before building a Spring Boot app, you should understand build tools and choose an IDE. This lesson explains Maven and Gradle in simple words, compares them with examples, recommends an IDE for beginners, and shows how to install Java, Maven, Gradle, and an IDE.

Maven Gradle IntelliJ IDEA Spring Boot Installation

1. What are Maven and Gradle?

Maven and Gradle are build tools. They help your Java project do important jobs such as:

  • download libraries automatically
  • compile Java code
  • run tests
  • package your app into a jar
  • run project tasks in a standard way
Easy idea: a build tool is like the project manager for your code. Instead of doing everything manually, you write rules once, and the tool handles the rest.

2. Maven explained

Maven is older, very common in Java projects, and uses an XML file named pom.xml. Many beginners find Maven easier to understand at first because its structure is more fixed and predictable.

Simple Maven dependency example

<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
</dependencies>

Simple Maven plugin example

<build>
    <plugins>
        <plugin>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-maven-plugin</artifactId>
        </plugin>
    </plugins>
</build>

Common Maven commands

mvn clean
mvn compile
mvn test
mvn spring-boot:run
mvn clean package

In Spring Boot starter projects, Maven usually creates a jar file and can run the application directly.

3. Gradle explained

Gradle is another build tool. It is flexible and modern, and supports build files written in Groovy or Kotlin DSL. In many projects, Gradle feels shorter and cleaner than Maven.

Simple Gradle dependency example (Groovy DSL)

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
}

Simple Gradle plugin example

plugins {
    id 'java'
    id 'org.springframework.boot' version '3.2.5'
}

Common Gradle commands

gradle clean
gradle build
gradle test
gradle bootRun
Important: Gradle officially recommends using the Gradle Wrapper (gradlew or gradlew.bat) so the project uses the correct Gradle version automatically.

Preferred wrapper commands

./gradlew build
./gradlew bootRun
gradlew.bat bootRun

4. Maven vs Gradle

Point Maven Gradle
Build file pom.xml build.gradle or build.gradle.kts
Style More structured, convention-heavy More flexible and programmable
Beginner readability Often easier at the start Can feel simpler later, but syntax varies
Customization Good Very strong
Spring Boot learning Excellent Excellent
Best beginner choice: start with Maven for your first few Spring Boot lessons. Its project structure is easier to teach and easier to debug when you are new. Learn Gradle after you are comfortable with dependencies, plugins, and packaging.

5. Which IDE is best?

For Spring Boot learning, the most balanced choice for most beginners is IntelliJ IDEA.

IDE Best for Notes
IntelliJ IDEA Most beginners and most Spring Boot learners Excellent code help, project wizard, very smooth Java experience
VS Code Lightweight editing and users who already like VS Code Works well with Java and Spring extensions, but setup feels more extension-driven
Eclipse Traditional Java users and some classroom environments Still capable, especially with Spring Tools, but many beginners find IntelliJ more comfortable

My recommendation

  • Start with IntelliJ IDEA Community Edition if you are learning Spring Boot.
  • Use VS Code only if you strongly prefer a lighter editor.
  • Use Eclipse if your college or company already uses it heavily.

6. Installation order

  1. Install JDK first
  2. Install an IDE
  3. Install Maven if you are using Maven outside the IDE
  4. Install Gradle only if you need system-wide Gradle; otherwise wrapper is enough
For Spring Boot starter projects, you often do not need a full global Gradle install if the project includes the Gradle Wrapper.

7. Install Java (JDK)

Java must be installed before Maven or Gradle can run properly. JDK 17 is a safe and common learning choice for Spring Boot.

Check Java after installation

java -version
javac -version

If these commands show installed versions, Java is ready.

8. Install Maven

Download the binary zip, extract it, and add Maven's bin folder to your PATH.

Basic Windows-style idea

  1. Download Maven zip
  2. Extract it to a folder such as C:\Tools\apache-maven-3.9.14
  3. Add C:\Tools\apache-maven-3.9.14\bin to PATH
  4. Open a new terminal and run:
mvn -version

If Maven is installed correctly, it will show the Maven version, Java version, and system details.

9. Install Gradle

If you want a global Gradle installation, download the binary zip, extract it, and add the Gradle bin folder to PATH. However, for most Gradle projects, the wrapper is the preferred method.

Check Gradle

gradle -version
The Gradle documentation says the Wrapper is the preferred way to start a Gradle build.

10. Install IntelliJ IDEA

Download IntelliJ IDEA Community Edition or Ultimate Edition from JetBrains. Community Edition is enough for core Java learning, and many beginners start there comfortably.

After installation

  • Open IntelliJ IDEA
  • Create or open a project
  • Make sure a JDK is configured
  • Import Maven or Gradle projects when prompted

11. Tiny examples: same idea in Maven and Gradle

Maven project idea

<project>
  <dependencies>
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
  </dependencies>
</project>

Gradle project idea

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
}

Both examples do the same thing: they tell the project to use Spring Boot Web. The difference is only the build tool syntax.

12. Best starting path for you

  1. Install JDK 17
  2. Install IntelliJ IDEA
  3. Install Maven
  4. Create a Spring Boot project using Maven + Spring Web
  5. Run a Hello World app
  6. After that, try the same project with Gradle
