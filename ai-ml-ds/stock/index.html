1) What we‚Äôre building

Goal: show a continuously updating chart of a stock price.
Key idea: never let fetching block the UI. Fetch in a background thread, send results via a queue, animate in the main thread.

System overview (mental model)

  • Fetcher (Selenium) reads latest price from Yahoo Finance.
  • Worker thread runs every N seconds, fetches price, pushes it into a Queue.
  • UI animation pulls latest values from the queue and updates the plot.
  • Windowed chart shows only last window_points samples using a deque(maxlen=...).
Why the Queue matters: Matplotlib UI runs on the main thread. Selenium is slow. The Queue is the safe ‚Äúbridge‚Äù between them.

D) test.py ‚Äî adapter + runner

This file is the glue that connects ‚Äúuser input stock code‚Äù ‚Üí ‚Äúprice fetcher function‚Äù ‚Üí ‚Äúlive chart engine‚Äù. The best part is the closure-based adapter: 

def make_fetchStockPrice(stockname: str):
    stockname = stockname.lower().strip()

    def fetchStockPrice() -> float:
        res = ut.getStockPrice(stockname)
        if isinstance(res, tuple) and len(res) >= 1:
            return float(res[0])
        if isinstance(res, Exception):
            raise res
        raise RuntimeError(f"Unexpected getStockPrice return: {res!r}")

    return fetchStockPrice

Why this pattern is excellent:

  • graphutilities expects a no-argument fetch_price() function.
  • But we want to pass a stockname too.
  • The closure ‚Äúcaptures‚Äù the stockname once, and exposes a clean no-argument function.
This is the key design win: your chart code stays generic, while your fetching code stays customizable.

3) How to run this locally

  • Install dependencies: pip install selenium matplotlib
  • Make sure Chrome is installed.
  • Make sure ChromeDriver is compatible with your Chrome version (or Selenium Manager handles it).
  • Run: python test.py
  • Enter a stock key: ril or th
Note: Yahoo Finance can change CSS selectors or block automation. If price stops fetching, check the saved HTML file (stockname_date.html) and update the selector.

4) Design decisions explained (why we did it this way)

‚úÖ Separation of concerns

  • Fetch logic lives in utilities.py
  • Chart logic lives in graphutilities.py
  • Mapping lives in stockslist.py
  • Wiring lives in test.py

‚úÖ Threading for responsiveness

Selenium page loads are slow. If you fetch on the UI thread, the graph freezes. So we fetch in a daemon worker thread.

‚úÖ Queue for thread safety

Never update Matplotlib objects from a background thread. The background thread should only push data into a queue. The main thread reads from the queue and updates the plot.
